# Cyber Security Learning Project

## 📚 О проекте

Этот репозиторий — мой учебный проект, в котором я изучаю **кибербезопасность, offensive security и разработку безопасных веб‑приложений**.

В проекте я:

* изучаю и реализую различные **техники атак**
* пишу собственные **инструменты для анализа сети**
* создаю **веб‑сайт на Django**, который позже буду использовать для тестирования уязвимостей

Основная цель проекта — **практическое изучение кибербезопасности через код**.

---

# 🎯 Цели проекта

* Изучить основы **offensive security**
* Реализовать популярные инструменты из области pentesting
* Научиться **искать и понимать уязвимости**
* Создать **собственную тестовую платформу на Django**
* Практиковаться в анализе сетевого трафика и логов

---

# 🧰 Реализованные инструменты

## 🔎 Port Scanner

Сканер портов, написанный на Python.

Функции:

* сканирование TCP‑портов
* определение открытых портов
* работа с сокетами
* понимание процесса TCP‑соединения

Используется для изучения:

* сетевых сервисов
* принципов работы TCP/IP
* методов разведки в pentest.

---

## 🐚 Reverse Shell

Простая реализация reverse shell на Python.

Используется для понимания:

* удалённого выполнения команд
* принципов работы shell‑соединений
* механизмов управления удалённой системой

⚠️ Используется **только в учебных целях**.

---

# 🌐 Django веб‑сайт

В проекте также создаётся веб‑сайт на Django, который будет использоваться как **платформа для изучения веб‑безопасности**.

## Функциональность сайта

Планируемые модули:

* 📰 Новости кибербезопасности
* 📚 База знаний
* 🛠 Каталог инструментов безопасности
* 🧪 Attack Lab (лаборатория атак)
* 🗺 Roadmap по кибербезопасности
* 📝 CTF writeups
* 📊 Анализ логов

Позже сайт будет использоваться для тестирования:

* XSS
* SQL Injection
* CSRF

* Broken access control

---

# 🧪 Attack Lab

Отдельный модуль для практики атак.

Здесь будут реализованы:

* xss_reflected_form — поле q, вывод значения на страницу. Поддержка режима safe/unsafe.
* sqli_login_form — логин-форма с небезопасным SQL (только в sandbox-вью с предупреждением).
* csrf_transfer_form — демонстрация CSRF на фейковой форме «перевод средств».
* upload_form — загрузка файлов, демонстрация insecure file upload и whitelist-проверок.
* idor_profile_form — ввод user_id для проверки object-level access control

---

# 🧪 Accounts

accounts — аутентификация и Broken Access Control

Базовые вью для логина/профиля, которые можно развить в учебный auth-блок.

Формы и сценарии:

* Login form — обычная форма + «лабораторная» версия с антипримером rate limiting/lockout.

* Change password form — проверка текущего пароля, enforce сложности.

* Profile edit form — демонстрация mass assignment и safe whitelist-полей

---

# 🧪 Knowledge_base

Формы CRUD для статей и комментариев, полезные для изучения stored XSS.

Формы:

* ArticleCreateForm / ArticleCommentForm:

  * unsafe режим: рендер HTML как есть (для демонстрации XSS),

  * safe режим: sanitize/escape для безопасного отображения

---

# 🧪 Main

reflected XSS и SSRF-подобные сценарии

Основные страницы со списками и деталями новостей.

Формы:

* Search form (search?q=...):

  * режим 1: корректный escaping,

  * режим 2 (lab): намеренно небезопасный вывод query

---

# 🧪 logs_analysis
загрузка и парсинг логов

Учебный модуль для работы с «грязными» входными данными.

Формы:

* Загрузка лог-файлов + выбор парсера.

Демонстрации:

* отсутствие ограничений размера/типа файла,

* regex DoS на невалидных строках,

* безопасный лимитированный парсинг

# 🚀 Запуск проекта

### 1. Клонировать репозиторий

```
git clone https://github.com/nikewalce/cyber_security.git
cd cyber_security
```

### 2. Создать виртуальное окружение

```
python -m venv .venv
```

Активировать:

Windows

```
.venv\Scripts\activate
```

Linux / Mac

```
source .venv/bin/activate
```

---

### 3. Установить зависимости

```
pip install -r requirements.txt
```

или

```
poetry install
```

---

### 4. Применить миграции

```
python manage.py migrate
```

---

### 5. Запустить сервер

```
python manage.py runserver
```

Сайт будет доступен по адресу:

```
http://127.0.0.1:8000
```
