📚 XSS (справочник + практика)

💥 Что такое XSS?

XSS (Cross-Site Scripting) — это уязвимость, при которой злоумышленник внедряет JavaScript в страницу.

Браузер НЕ понимает, что это атака → и выполняет код.

🧪 Типы XSS

  • Reflected → приходит из URL и сразу выполняется
  • Stored → сохраняется в базе и выполняется у всех
  • DOM → происходит полностью в JavaScript

⚠️ Почему это опасно?

  • Кража cookie (сессии)
  • Подмена страницы
  • Фишинг внутри сайта
  • Действия от имени пользователя

🍪 Cookie и XSS

Атака:

<script>
alert(document.cookie)
</script>

Если cookie доступны → злоумышленник может украсть сессию.

Защита:

  • HttpOnly → JavaScript НЕ может читать cookie

🛡️ Как работает CSP

Сервер отправляет заголовок:

Content-Security-Policy: ...

Браузер читает его и говорит:

"Я буду выполнять только разрешённый код"

Почему это важно:

  • Даже если XSS есть → JS может НЕ выполниться
  • Можно запретить inline-скрипты
  • Можно ограничить источники

🧰 Режимы лаборатории

  • Vulnerable → XSS работает
  • Secure → Django autoescape / forms
  • Super Secure → принудительное экранирование (escape)
  • CSP ON → браузер блокирует JS

🔥 Попробуй атаки прямо сейчас!

Готовые payload'ы:

  • <script>alert(1)</script>
  • <img src=x onerror=alert(1)>
  • <svg onload=alert(1)>
  • <script>alert("Cookie: " + document.cookie))</script>

🎯 Задачи для практики:

  • Включи CSP и попробуй payload
  • Сравни vulnerable vs secure режимы
  • Проверь cookie через кнопку выше
